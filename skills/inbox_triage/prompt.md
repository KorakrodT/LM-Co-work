# Workflow: คัดกรองไฟล์ inbox

ทำตามขั้นตอนนี้ตามลำดับ:

1. เรียก `list_files("inbox")` — ถ้าโฟลเดอร์ว่างหรือไม่มี ให้แจ้งผู้ใช้แล้วจบ
2. อ่านไฟล์ข้อความแต่ละไฟล์ด้วย `read_file` (ข้ามไฟล์ binary — ดูจากนามสกุล)
3. จัดแต่ละไฟล์เข้า 3 กลุ่ม พร้อมเหตุผลสั้นๆ:
   - **ต้องทำต่อ** → ควรย้ายไป projects/ (งานที่มี action ค้างอยู่)
   - **เก็บอ้างอิง** → ควรย้ายไป archive/ (จบแล้ว/แค่เก็บไว้ดู)
   - **ถามผู้ใช้** → ตัดสินไม่ได้จากเนื้อหา
4. แสดงผลเป็นตาราง: ไฟล์ | กลุ่ม | เหตุผล | สิ่งที่ควรทำต่อ
5. **อย่าย้าย/ลบไฟล์เอง** — เสนอรายการให้ผู้ใช้ตัดสินใจ ถ้าผู้ใช้สั่งย้าย ให้ใช้ `read_file`
   + `write_file` สร้างสำเนาที่ปลายทางแล้วแจ้งให้ผู้ใช้ลบต้นฉบับเอง

กติกา: ห้ามสรุปเนื้อหาไฟล์เกินจริง ไฟล์ไหนอ่านไม่ได้ให้ใส่กลุ่ม "ถามผู้ใช้"
