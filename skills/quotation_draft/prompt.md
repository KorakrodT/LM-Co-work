# Workflow: ร่างใบเสนอราคา / ใบแจ้งหนี้

ทำตามขั้นตอนนี้ตามลำดับ:

1. เช็คก่อนว่าใน `_memory.md`/บริบทมีข้อมูลผู้ขาย (ชื่อกิจการ, ที่อยู่, เลขผู้เสียภาษี,
   ธนาคาร/เลขบัญชี) อยู่แล้วหรือไม่ — มีแล้วห้ามถามซ้ำ
2. ถามข้อมูลที่ขาดจากผู้ใช้ครั้งเดียวจบ:
   - ประเภทเอกสาร (ใบเสนอราคา หรือ ใบแจ้งหนี้), ชื่อ-ที่อยู่ลูกค้า
   - รายการสินค้า/บริการ: ชื่อ, จำนวน, ราคาต่อหน่วย
   - ส่วนลด (ถ้ามี), VAT 7% หรือไม่, กำหนดยืนราคา/ชำระเงิน
3. คำนวณด้วย `calculator` ทีละขั้น: ยอดรวม → หักส่วนลด → VAT → ยอดสุทธิ
   (ห้ามคำนวณในหัว — ตัวเลขเงินผิดไม่ได้)
4. จัดรูปแบบเอกสาร: หัวเอกสาร (เลขที่ QT/INV-YYYYMMDD-01 + วันที่จาก `get_current_time`),
   ข้อมูลผู้ขาย/ลูกค้า, ตารางรายการ, สรุปยอด, เงื่อนไข, ช่องลงชื่อ
5. เสนอบันทึกด้วย `write_file` ชื่อ `ใบเสนอราคา-<ลูกค้า>-<วันที่>.md`
6. ถ้าผู้ใช้ให้ข้อมูลผู้ขายมาใหม่ในแชตนี้ → ใช้ `remember` เสนอจดไว้ จะได้ไม่ต้องถามครั้งหน้า

กติกา: ตัวเลขทุกตัวต้องมาจาก calculator เท่านั้น แสดงทศนิยม 2 ตำแหน่ง
