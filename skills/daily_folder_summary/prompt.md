# Workflow: สรุปโฟลเดอร์งานประจำวัน

ทำตามขั้นตอนนี้ตามลำดับ:

1. เรียก `list_files` ที่รากโฟลเดอร์งาน แล้วเรียก `list_files("inbox")` และ `list_files("projects")` (ถ้ามี)
2. ถ้ามีไฟล์ใน inbox/: อ่านไฟล์ข้อความด้วย `read_file` ทีละไฟล์ (ข้ามไฟล์ binary เช่นรูป/zip)
3. สรุปเป็นรายงานสั้นภาษาไทย 3 ส่วน:
   - **ไฟล์ใหม่ใน inbox** — แต่ละไฟล์คืออะไร ควรทำอะไรต่อ (1 บรรทัด/ไฟล์)
   - **งานที่ค้างใน projects** — เห็นอะไรน่าตามต่อ
   - **สิ่งที่ควรทำวันนี้** — 3 ข้อเรียงตามความสำคัญ
4. เสนอบันทึกรายงานด้วย `write_file` ชื่อ `รายงานประจำวัน-<วันที่ YYYY-MM-DD>.md` (ใช้ `get_current_time` หาวันที่)

กติกา: ตอบกระชับ ไม่เกิน 20 บรรทัด อย่าเดาเนื้อหาไฟล์ที่อ่านไม่ได้ — บอกตรงๆ ว่าข้าม
